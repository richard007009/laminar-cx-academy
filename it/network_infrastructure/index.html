Network Models and Standards

A solid understanding of networking begins with the reference models that define how data travels from one device to another across a network.

The OSI Reference Model

TCP/IP Model

TCP/IP Four-Layer Model:

  • Application Layer: End-user protocols (HTTP, DNS, DHCP, SMTP)
  • Transport Layer: TCP (reliable) and UDP (connectionless) communication
  • Internet Layer: IP addressing, routing, and packet forwarding
  • Network Access Layer: Physical transmission over media (Ethernet, Wi-Fi)

📋 Key Standards Bodies

  • 📌 IEEE — Ethernet (802.3), Wireless LAN (802.11), VLAN (802.1Q)
  • 📌 IETF — TCP/IP suite, routing protocols (RFCs)
  • 📌 ANSI/TIA — Structured cabling standards (TIA-568)
  • 📌 ISO/IEC — Network management and cabling (ISO 11801)
  • 📌 ITU-T — Telecommunications and WAN standards

Network Topologies and Architecture

Network topology defines the physical or logical arrangement of devices and connections. Choosing the right topology is critical for performance, redundancy, and scalability.

Common Network Topologies

Enterprise Three-Tier Architecture

Layer Role Key Devices Key Feature
Core High-speed backbone Core switches, routers Maximum throughput
Distribution Policy and aggregation Layer 3 switches Routing, redundancy
Access End-device connectivity Access switches, APs Port security, PoE

Modern Alternatives:

  • Collapsed Core: Two-tier design for smaller environments
  • Spine-Leaf: Low-latency data center fabric architecture
  • SD-WAN: Software-defined wide area networking for branch offices
  • Cloud-First: Hybrid architectures leveraging public cloud interconnects

Core Network Hardware

Understanding the role of each hardware component is essential for designing, deploying, and troubleshooting enterprise networks.

Primary Network Devices

Switches (Layer 2 / Layer 3):

  • Layer 2 Switch: Forwards frames using MAC address tables; creates collision domains per port
  • Layer 3 Switch: Performs inter-VLAN routing at wire speed; replaces routers in campus LANs
  • Managed vs. Unmanaged: Managed switches support VLANs, SNMP, STP, QoS, and port mirroring
  • PoE Switches: Provide power over Ethernet to IP phones, cameras, and wireless APs

Routers:

  • Function: Route packets between different IP networks using routing tables
  • Protocols: OSPF, BGP, EIGRP, IS-IS for dynamic route exchange
  • Edge Routers: Connect enterprise LAN to ISP/WAN; typically run NAT and ACLs
  • Core Routers: High-capacity devices used in ISP backbones and large DCs

Supporting Infrastructure

Device Layer Function Common Use
Firewall 3–7 Traffic inspection & policy enforcement Perimeter, DMZ, segmentation
Load Balancer 4–7 Distribute traffic across servers Web farms, application tiers
Wireless AP 1–2 802.11 wireless access Office, warehouse, campus

⚠️ Hardware Selection Considerations:

  • Always check port density, throughput capacity, and backplane bandwidth
  • Ensure hardware supports required standards (IEEE 802.3, 802.11ax, etc.)
  • Confirm EOL/EOS timelines before procurement
  • Plan for N+1 redundancy on critical switching and routing equipment

IP Addressing and Subnetting

IP addressing is the foundation of network communication. Every device on a network requires a unique IP address to send and receive data.

IPv4 Address Classes

Class Range Default Subnet Typical Use
A 1.0.0.0 – 126.255.255.255 /8 (255.0.0.0) Large enterprises, ISPs
B 128.0.0.0 – 191.255.255.255 /16 (255.255.0.0) Medium-large organizations
C 192.0.0.0 – 223.255.255.255 /24 (255.255.255.0) Small networks

RFC 1918 Private Address Space

Reserved Private Ranges (Non-routable on internet):

  • 10.0.0.0/8 — Up to 16,777,216 hosts; used in large enterprises and data centers
  • 172.16.0.0/12 — Up to 1,048,576 hosts; common in mid-size environments
  • 192.168.0.0/16 — Up to 65,536 hosts; common in branch offices and home networks

CIDR and Subnetting

IPv6 Overview

Why IPv6 Matters:

  • Address Space: 128-bit addresses — 340 undecillion unique addresses
  • No NAT Required: Every device can have a globally routable address
  • Auto-configuration: SLAAC enables stateless address assignment
  • Dual-Stack: Transitional environments run IPv4 and IPv6 simultaneously
  • Security: IPSec support is built into the IPv6 specification

LAN and VLAN Design

Proper LAN design ensures performance, security, and manageability across the enterprise. VLANs are a fundamental tool for network segmentation.

VLAN Fundamentals

What VLANs Provide:

  • Broadcast Domain Control: Limit broadcast traffic to within a VLAN segment
  • Security Segmentation: Isolate sensitive traffic (HR, Finance, IoT) from general users
  • Flexibility: Group users logically regardless of physical location
  • Simplified Management: Apply consistent QoS and ACL policies per VLAN

Common VLAN Assignments

VLAN ID Name Typical Use Notes
1 Default Native/untagged (avoid in production) Security risk if left as native
10 Management Switch/router OOB management Restrict access via ACL
20 Data / Users General workstation traffic DHCP-assigned
30 Voice VoIP phones Requires QoS (EF DSCP)
40 Servers Internal application servers Tighten firewall rules
50 Guest / IoT Visitor Wi-Fi, IoT devices Internet-only, no internal access

Spanning Tree Protocol (STP)

⚠️ STP and Loop Prevention:

  • STP (802.1D): Classic spanning tree; slow convergence (30–50 seconds)
  • RSTP (802.1w): Rapid STP; typical convergence under 6 seconds
  • MSTP (802.1s): Multiple spanning tree for VLAN-based load balancing
  • Best Practice: Always configure root bridge placement; enable PortFast on access ports

Trunking and 802.1Q:

  • Trunk ports carry multiple VLANs between switches using 802.1Q tagging
  • Always change the native VLAN away from VLAN 1 to prevent VLAN hopping attacks
  • LACP (802.3ad) Link Aggregation can bundle multiple trunk links for bandwidth and redundancy

WAN and Connectivity Technologies

Wide Area Networks connect geographically dispersed sites. Choosing the right WAN technology balances cost, bandwidth, latency, and reliability requirements.

VPN Technologies

Common VPN Types:

  • IPSec Site-to-Site: Encrypted tunnel between two fixed endpoints; common for branch connectivity
  • SSL/TLS Remote Access: Client-based VPN for remote workers (e.g., Cisco AnyConnect, GlobalProtect)
  • DMVPN: Dynamic Multipoint VPN; scalable hub-and-spoke with spoke-to-spoke capability
  • ZTNA: Zero Trust Network Access; replaces traditional VPN with identity and context-aware access

⚠️ WAN Design Considerations:

  • Always provision redundant WAN circuits from diverse carriers and physical paths
  • Calculate bandwidth requirements including peak usage, video conferencing, and cloud application traffic
  • Implement QoS policies to prioritize voice and critical application traffic on congested links
  • Define RPO and RTO targets for WAN failover in your BCP/DRP documentation

Wireless Networking (WLAN)

Wireless networks are now a primary access method in enterprise environments. A well-designed WLAN provides seamless, secure, and reliable connectivity across the facility.

WLAN Security Standards

Authentication and Encryption:

  • WPA3-Enterprise: 192-bit security suite; mandatory for high-security environments
  • WPA3-Personal: SAE replaces PSK; resistant to offline dictionary attacks
  • 802.1X/EAP: RADIUS-based user/device authentication for enterprise WLANs
  • Avoid WEP/WPA: Legacy protocols are cryptographically broken; do not use

RF Planning Principles

Site Survey Best Practices:

  • Conduct predictive RF planning before deployment using heat-mapping tools
  • Perform a post-installation active survey to verify signal strength (>-67 dBm for voice/video)
  • Use non-overlapping channels: 1, 6, 11 on 2.4 GHz; auto-channel on 5/6 GHz
  • Deploy in controller or cloud-managed mode for seamless roaming (802.11r/k/v)
  • Separate corporate SSID from guest using dedicated VLANs and firewall rules

Network Security Fundamentals

Network security is not an add-on — it must be designed into the infrastructure from the ground up. A layered defense-in-depth strategy reduces risk across all network tiers.

Defense-in-Depth Architecture

Key Security Controls

Network Access Control (NAC):

  • 802.1X Port Authentication: Verify identity before granting LAN/WLAN access
  • RADIUS/TACACS+: Centralized AAA (Authentication, Authorization, Accounting)
  • MAC Address Filtering: Basic control; can be bypassed — use as additional layer only
  • Posture Assessment: Verify endpoint compliance before network admission

🚨 Common Network Attack Vectors:

  • VLAN Hopping: Switch spoofing or double-tagging — mitigate with native VLAN best practices
  • ARP Spoofing: Redirect traffic via forged ARP replies — enable Dynamic ARP Inspection (DAI)
  • DHCP Starvation: Exhaust the DHCP pool — enable DHCP Snooping on access switches
  • Rogue APs: Unauthorized wireless access points — use WIDS/WIPS detection

Network Management and Monitoring

Effective network management ensures visibility, control, and rapid response to incidents. A combination of protocols and tools provides comprehensive operational insight.

NTP UDP 123 Clock synchronization Stratum 2 internal NTP

Monitoring and NOC Tooling

Essential Monitoring Capabilities:

  • Availability Monitoring: ICMP/SNMP ping checks with alerting thresholds for uptime SLAs
  • Performance Dashboards: Interface utilization, error rates, CPU/memory on network devices
  • Bandwidth Analysis: NetFlow-based top-talker and application identification
  • Configuration Management: Automated backup and change tracking (e.g., RANCID, Oxidized)
  • Network Topology Maps: Auto-discovered maps using LLDP/CDP and SNMP MIB data

Documentation Requirements:

  • Maintain up-to-date network diagrams (Layer 1, Layer 2, Layer 3) in a change-controlled system
  • Document IP address management (IPAM) — use dedicated tools for large environments
  • Keep hardware inventory records including serial numbers, firmware versions, and support contracts
  • Record all configuration changes with timestamps and change ticket references

Cabling and Physical Infrastructure

The physical layer is the foundation of every network. Structured cabling standards ensure a reliable, scalable, and maintainable cable plant.

Copper Cabling Standards (TIA-568)

Category Max Speed Max Distance Use Case
Cat 5e 1 Gbps 100 m Legacy enterprise
Cat 6 1 Gbps / 10 Gbps* 100 m / 55 m* Standard desktop access
Cat 6A 10 Gbps 100 m Access layer, PoE+ APs
Cat 8 25/40 Gbps 30 m Top-of-rack DC links

Fiber Optic Cabling

Fiber Types and Applications:

  • OM3/OM4 Multimode: 10/40/100 Gbps within campus (up to 400 m); lower cost transceivers
  • OM5 Multimode: SWDM support for higher bandwidth over short distances
  • OS2 Single-mode: Long-distance links (metro, inter-building); supports DWDM up to 80 km+
  • Pre-terminated MPO/MTP: High-density fiber trunks for data center structured cabling

MDF / IDF Best Practices:

  • Maintain 1:3 ratio of patch panel ports to active ports for future growth
  • Label every cable, port, and panel with a consistent naming convention
  • Separate power and data cabling to avoid EMI interference
  • Use horizontal cable managers and velcro straps — never zip ties on active cables
  • Conduct cable certification testing and retain test reports for warranty compliance

Compliance, Standards, and Documentation

Network infrastructure must align with applicable compliance frameworks and industry standards. Proper documentation is the foundation of audit readiness and operational continuity.

Relevant Compliance Frameworks

Common Frameworks Affecting Network Design:

  • PCI DSS: Cardholder data environments require network segmentation, firewall rule reviews, and quarterly scans
  • HIPAA: PHI transmission requires encryption in transit; audit logging of access to healthcare systems
  • ISO 27001: Information security management; requires documented network controls and risk assessments
  • NIST CSF: Identify, Protect, Detect, Respond, Recover framework for cybersecurity risk management
  • SOC 2 Type II: Trust service criteria including availability and confidentiality of networked systems
